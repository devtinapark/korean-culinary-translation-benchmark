Evaluation Notes

DimensionFinding
Schema completeness Claude is the only model to achieve 1.000 on all EN scenarios. EN schema is harder — more optional fields left null. KO schemas are consistently fuller across all models.
Loanword register All three models output English "sesame oil" in some form for EN scenarios. Claude used 세서미 오일 (phonetic Konglish), but this was a system-prompt artifact — no source transcript contains this form. The correct native Korean term is 참기름. The system prompt has been updated to remove 세서미 오일 as an example and clarify that 참기름 is not a loanword.
KO loanword unanimity All models score 1.000 on KO loanword preservation. Korean-script loanwords (프라이팬, 레시피) are visually distinct and consistently present in output text, even when absent from loanwords_detected.
Noise robustness Gemini shows the smallest schema degradation under noise (Δ −0.012). Qwen marginally improves on noisy KO (+0.006), likely due to extended-thinking engagement on ambiguous input.
Qwen language gap Qwen's EN schema (0.847) is 0.135 below its KO schema (0.983) — the largest language gap. A KO-only benchmark would significantly overstate Qwen's multilingual capability.
Cultural score Stub at 0 for all models. Activating the judge LLM is the highest-leverage change for the next run; qualitative reading of hidden_intent and cultural_notes in the CSVs suggests Claude would lead this axis.

Bug Log — Metric Inversion in src/benchmark.py

Root cause

The original BenchmarkResult.to_dict() passed raw avg_schema_validity (higher-is-better) directly into the "cer" slot of ModelRanker: 

# Bug — code that produced the original results.csv
"cer": self.avg_schema_validity,        # ← no inversion
"wer": self.avg_cultural_score / 5.0,   # ← no inversion

ModelRanker.normalize_metric() at ranking.py:166 normalizes cer with lower_is_better=True: 

norm = (v - min_val) / (max_val - min_val)
if lower_is_better:
    norm = 1.0 - norm   # lowest raw value → 1.0 (best rank)

Effect on each model

Schema validity inputs (raw, no inversion):
  Claude  = 0.9886  →  cer_norm = 0.000  ← penalized for highest schema
  Gemini  = 0.9350  →  cer_norm = 0.273
  Qwen    = 0.9149  →  cer_norm = 1.000  ← rewarded for lowest schema

Cultural WER = 0.0 for all → wer_norm = 0.5 for all (tie; no differentiation)

Loanword (correctly normalized, lower_is_better=False):
  Gemini  = 0.8882  →  loanword_norm = 1.000
  Claude  = 0.8859  →  loanword_norm = 0.957
  Qwen    = 0.8352  →  loanword_norm = 0.000

Buggy composite (0.40·cer_norm + 0.25·wer_norm + 0.35·loanword_norm):
  Gemini  → 0.109 + 0.125 + 0.350 = 0.7658  rank 1
  Qwen    → 0.400 + 0.125 + 0.000 = 0.5250  rank 2
  Claude  → 0.000 + 0.125 + 0.335 = 0.4600  rank 3

Correct formula (raw per-scenario CSV averages)

Correct composite = 0.40 × schema + 0.35 × loanword + 0.25 × cultural
  Claude  → 0.3954 + 0.3101 + 0 = 0.7055  rank 1
  Gemini  → 0.3740 + 0.3109 + 0 = 0.6849  rank 2
  Qwen    → 0.3660 + 0.2923 + 0 = 0.6583  rank 3

Fix (current src/benchmark.py)

"cer": 1.0 - self.avg_schema_validity,          # lower = more valid ✓
"wer": 1.0 - (self.avg_cultural_score / 5.0),   # lower = higher judge score ✓
"loanword_accuracy": self.avg_loanword_score,    # already higher-is-better ✓

Run python -m src.main to regenerate results.csv with correct rankings.
